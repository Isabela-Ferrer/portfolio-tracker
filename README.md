# Portfolio Tracker

An AI-powered momentum dashboard for venture capital analysts. Track portfolio companies across press, hiring, product launches, funding, app store ratings, and community signals — all in one place, refreshed on demand.

---

## What it does

For each company in your portfolio, the tracker automatically gathers signals from multiple sources, scores them, and surfaces the three most VC-relevant insights using GPT-4o-mini. No spreadsheets. No manual Googling.

**Dashboard view** — all companies ranked by momentum score with signal columns:

| Signal | What it tracks |
|---|---|
| 📰 Press highlights | Top 3 AI-extracted bullet points from recent news |
| 💼 Jobs | Open headcount + hiring velocity vs. prior snapshot |
| ⭐ App Store | Average rating across iOS and Android |
| 🚀 Launches | Recent Product Hunt posts and GitHub releases |
| 💰 Funding | Latest round — amount, type, and date |
| 🟠 Reddit | Recent posts discussing the company |
| ⚡ Key signals | 3 VC-relevant bullets generated by GPT-4o-mini |

**Company detail view** — full signal breakdown with trends over time.

---

## Scoring

Each company gets a **momentum score (0–100)** based on weighted signals:

| Signal | Weight | How it's calculated |
|---|---|---|
| Jobs | 30% | 5 pts per open role (max 60) + 40 pt growth bonus if headcount increased |
| Press | 25% | 10 pts per article + 15 pt bonus for Tier 1 outlets (TechCrunch, Bloomberg, etc.) |
| App Store | 20% | Rating normalized to 100 (4.0★ → 80 pts) |
| Launches | 20% | 50 pts per launch in the last 60 days |
| Funding | 5% | 100 pts if any funding event in the last 120 days, else 0 |

Trend arrows (↑ ↓ →) show how each signal has moved vs. prior snapshots.

## Architecture

```
portfolio tracking/
├── main.py               # FastAPI app — routes and startup
├── orchestrator.py       # Runs all fetchers concurrently, scores, persists
├── scorer.py             # Weighted signal scoring logic
├── models.py             # Dataclasses: Company, PressArticle, FundingSignal, etc.
├── database.py           # SQLite — schema, migrations, read/write helpers
├── ai_narrator.py        # GPT-4o-mini: generates AI blurb + 3 VC-signal bullets
├── ai_extractor.py       # GPT-4o-mini: enriches press key points + funding details
├── trend_calculator.py   # Computes delta trends across historical snapshots
├── fetchers/
│   ├── press.py          # Google News RSS → relevance-filtered press articles
│   ├── funding.py        # Google News RSS → funding signals with AI enrichment
│   ├── jobs.py           # Job board scraping → open roles + headcount
│   ├── appstore.py       # App Store + Play Store ratings and reviews
│   ├── product_launches.py # Product Hunt + GitHub releases
│   └── reddit.py         # Reddit search → on-topic community posts
└── frontend/
    ├── index.html        # Dashboard — Alpine.js + Tailwind CSS
    ├── company.html      # Company detail page
    └── settings.html     # Add / edit / delete companies
```

**Stack:** Python 3.11 · FastAPI · SQLite · Alpine.js · Tailwind CSS · OpenAI GPT-4o-mini

## Setup

**1. Create and activate a virtual environment**

```bash
python -m venv .venv
source .venv/bin/activate
```

**2. Install dependencies**

```bash
pip install fastapi uvicorn httpx openai pydantic
```

**3. Set your OpenAI API key**

```bash
export OPENAI_API_KEY="sk-..."
```

Or add it to a `.env` file and load it with your preferred method.

**4. Start the server**

```bash
python -m uvicorn main:app --reload
```

**5. Open the dashboard**

Visit [http://localhost:8000](http://localhost:8000)

> The database (`data/portfolio.db`) is created automatically on first run. No migrations needed — the app handles schema updates at startup.

## Adding a company

Go to **Settings** (gear icon) and fill in:

| Field | Required | Notes |
|---|---|---|
| Company name | ✅ | Used for all news and Reddit searches |
| Website URL | ✅ | Used for logo, domain-based press filtering |
| App Store URL | — | iOS app link for rating tracking |
| Play Store URL | — | Android app link for rating tracking |
| GitHub org / repo | — | For tracking releases |
| Product Hunt slug | — | For tracking launches |
| G2 slug | — | For G2 review tracking |

After adding, the first refresh runs automatically in the background.

---

## Refreshing data

- **Single company:** Click the refresh button on the company card or detail page.
- **All companies:** Use the "Refresh All" button in the dashboard header.

Each refresh fetches all signals concurrently, re-scores, generates new AI bullets, and persists a new snapshot with a timestamp.

## Signal sources

| Signal | Source |
|---|---|
| Press | Google News RSS (two passes: domain-anchored + business-signal query) |
| Funding | Google News RSS filtered for funding keywords, AI-enriched for amount/round/investors |
| Jobs | Public job board APIs |
| App ratings | Apple App Store + Google Play Store |
| Launches | Product Hunt + GitHub Releases |
| Reddit | Reddit search RSS (exact-phrase + domain queries) |

**Relevance filtering:** Press and Reddit results are filtered with word-boundary regex to ensure articles and posts are genuinely about the company — not passing name mentions. Business-signal keywords (raises, product, platform, review, million, etc.) are required in the title or snippet for press articles to pass.
